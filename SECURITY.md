# Security Policy

## Supported Versions

Only the latest release of PostDesk is actively supported with security updates. We recommend always keeping your local application or container instances updated to the latest commit on the `main` branch.

| Version | Supported          |
| ------- | ------------------ |
| v1.0.x  | :white_check_mark: |
| < v1.0  | :x:                |

## Reporting a Vulnerability

We take the security of PostDesk and the safety of creators' storage databases very seriously. If you discover a vulnerability, please do **NOT** open a public GitHub issue. This exposes code vectors before they can be remediated.

Instead, please report security vulnerabilities directly and discretely:

- **Primary Channel:** Secure Message or DM on X (Twitter): [@milonspace](https://x.com/milonspace)
- **Secondary Channel:** Secure Message on Instagram: [@withmilon](https://www.instagram.com/withmilon)

Please include:
1. A clear description of the vulnerability vector.
2. Step-by-step instructions or test payloads to reproduce the issue locally.
3. Potential impact estimates (e.g. data tampering, credential exposure, cross-site leaks).

We will evaluate the threat, plan a patch, and release an update in a timely manner. Credit will be given in our release notes for responsible disclosures!
